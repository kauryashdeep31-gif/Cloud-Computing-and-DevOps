# Day 9 Summary (10/07/2026)

## Topics Covered

- Learned about **AWS Identity and Access Management (IAM)** and understood that it is a **global AWS service** used to securely manage access to AWS resources.
- Explored the core concepts of **authentication** and **authorization**, and understood how IAM helps control who can access specific AWS services.
- Created **IAM users**, configured login passwords, and learned how individual users can securely access the AWS Management Console.
- Studied **IAM Groups** and learned how grouping users simplifies permission management by assigning the same permissions to multiple users at once.
- Explored **IAM Policies** and understood how JSON-based policy documents define what actions users and groups are allowed or denied to perform.
- Learned about **AWS security credentials**, including usernames, passwords, and access keys, and understood their role in securely accessing AWS services.
- Explored **Multi-Factor Authentication (MFA)** and learned how adding an extra verification step significantly improves account security.
- Studied common cybersecurity threats such as **keyloggers** and **brute-force attacks**, and understood how AWS security features help mitigate these risks.
- Gained practical knowledge of AWS security best practices, including the **principle of least privilege**, using strong passwords, enabling MFA, and avoiding the use of the root account for daily tasks.

## Learning Outcomes

- Developed a strong understanding of AWS IAM and identity management.
- Learned how to create and manage IAM users, groups, and policies.
- Understood the importance of authentication, authorization, and secure access control.
- Gained knowledge of AWS security credentials and Multi-Factor Authentication (MFA).
- Learned how AWS security features help protect cloud resources from common cyber threats.
- Improved practical understanding of cloud security best practices and access management in AWS.
