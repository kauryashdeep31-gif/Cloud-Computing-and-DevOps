**Day 11 Summary (14-07-2026)**

Today's session focused on understanding the fundamentals of **Git** and **GitHub**, which are essential tools for version control and collaborative software development. I learned the key difference between the two: **Git** is a distributed version control system used to track changes in source code, whereas **GitHub** is a cloud-based platform that hosts Git repositories and enables collaboration among developers through features such as repositories, branches, pull requests, and code reviews.

I gained hands-on experience with several important Git commands that are commonly used in software development. These included **git init** for initializing a repository, **git add** for staging files, **git commit** for saving changes with meaningful commit messages, **git push** for uploading local changes to a remote repository, **git pull** for synchronizing the local repository with the latest remote updates, and **git branch** for creating and managing different branches. Practicing these commands helped me understand the complete workflow involved in tracking and managing project changes using Git.

As part of the practical session, I created **two Git repositories** and explored repository management through branching. I worked with both the **main** branch and a separate **feature** branch, learning how branches allow developers to work on new features independently without affecting the stable version of the project. This demonstrated the importance of branching in collaborative development and parallel feature implementation.

To apply these concepts, I developed and modified a simple **TechNova** website while working on the **feature** branch. After completing the required changes, I learned how to merge the feature branch into the **main** branch, understanding the purpose of branch merging and how it integrates new features into the primary codebase while maintaining project consistency.

The session also introduced me to **Pull Requests (PRs)** on GitHub. I created a Pull Request to merge changes from the feature branch into the main branch and learned how PRs facilitate collaboration by allowing code review and discussion before merging changes. Additionally, I explored the repository's **commit history**, which helped me understand how Git records every change made to a project, making it easier to track modifications, review previous versions, and maintain a clear development history.

Overall, today's session provided a strong foundation in Git and GitHub workflows, including repository creation, version control, branching strategies, branch merging, pull requests, and commit history management. These concepts are fundamental to collaborative software development and form an important part of modern DevOps and software engineering practices.
