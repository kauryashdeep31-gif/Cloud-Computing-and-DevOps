# Day 10 Summary (13/07/2026)

## AWS S3 Storage and Static Website Hosting

Today focused on understanding **Amazon S3 (Simple Storage Service)**, one of the most widely used storage services in AWS. Learned how Amazon S3 stores data as objects inside buckets and provides highly available, secure, and scalable cloud storage.

### Practical Activities Performed

- Studied the architecture and working of Amazon S3.
- Created a new S3 bucket following AWS bucket naming rules.
- Uploaded multiple files and explored object management features such as viewing, downloading, and deleting stored objects.
- Learned how bucket permissions work and understood the differences between private and publicly accessible buckets.
- Explored **S3 Versioning**, including how AWS stores multiple versions of the same object to prevent data loss and simplify file recovery.
- Configured **Static Website Hosting** on an S3 bucket by uploading website files, enabling website hosting, setting the index document, and accessing the hosted website through the S3 website endpoint.
- Understood real-world use cases of Amazon S3, including website hosting, backup storage, media storage, and application data storage.

## Key Learning Outcomes

- Developed a clear understanding of Amazon S3 architecture and object storage concepts.
- Learned to create and manage S3 buckets and objects effectively.
- Understood bucket security, permissions, and access control.
- Learned how versioning improves data reliability and recovery.
- Successfully hosted a static website using Amazon S3 and understood the basic deployment workflow.
